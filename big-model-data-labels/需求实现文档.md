# 需求实现文档

## 项目概述

**智能数据标注与分析平台** - 基于DeepSeek大模型的自动化数据标注系统

本平台允许用户通过自定义标签规则，利用DeepSeek大模型对 Excel/CSV 文件进行自动批量标注、人工修正和分析。

**技术栈：**
- 后端：Spring Boot 2.7 + Java 8
- 前端：Vue 3 + TypeScript + Element Plus
- 数据库：MySQL 8.0+
- 大模型：DeepSeek API (OpenAI兼容接口)
- 缓存：Redis 7+

---

## 1. 用户认证与权限管理

### 1.1 用户登录
- 支持用户名/密码登录
- JWT Token 认证机制
- Token 自动刷新

### 1.2 角色控制
- 管理员角色（admin）：拥有所有权限
- 普通用户角色（normal）：仅操作自己的数据

### 1.3 数据隔离
- 普通用户之间的数据、任务、标签完全隔离
- 所有数据查询包含用户身份过滤
- 管理员可查看所有用户数据

### 1.4 操作审计
- 记录用户关键操作日志
- 审计日志包含：操作人、操作类型、资源类型、资源ID、详细信息

### 2.4 数据集详情
- 查看数据集所有数据行
- 支持关键词搜索
- 分页加载大数据量

### 2.5 数据集归档
- 归档后数据变为只读
- 支持取消归档

### 2.6 外部数据库导入

## 3. 标签系统

### 3.1 标签类型
- **分类判断（classification）**：判断数据行是否符合标签规则，返回"是/否"
- **LLM提取（extraction）**：使用大模型从数据中提取指定信息
- **结构化号码提取（structured_extraction）**：基于正则表达式提取结构化数据（手机号、身份证等）

### 3.2 标签作用域
- **全局标签（global）**：所有数据集可用
- **数据集专属（dataset）**：仅指定数据集可用
- **任务临时（task）**：仅在当前分析任务中使用

### 3.3 版本管理
- 标签修改自动创建新版本
- 历史版本可追溯查看
- 分析任务绑定标签的具体版本
- 确保历史任务结果一致性

### 3.4 预处理模式
- **仅大模型（llm_only）**：直接调用大模型判断
- **仅规则（rule_only）**：仅使用提取器规则判断
- **规则后大模型（rule_then_llm）**：先规则匹配，匹配失败再调用大模型

### 3.5 二次强化分析
- 对低信心度结果进行二次分析
- 可配置触发条件（信心度阈值）
- 可配置不同的提示词模板

### 3.6 标签管理
- 创建、编辑、删除标签
- 批量激活/禁用标签
- 查看标签使用统计

## 4. AI 分析任务

### 4.1 任务创建
- 选择数据集
- 选择标签（支持多选）

### 4.3 进度追踪
- 实时显示处理进度百分比
- 显示已处理行数、总行数
- 显示失败行数

### 4.4 执行日志
- 记录任务执行过程日志
- 支持按日志级别筛选
- 实时推送最新日志

### 4.5 任务导出
- 导出任务结果为 Excel/CSV
- 包含原始数据和标签结果

## 核心数据模型

### 用户（User）
- id、username、password_hash、role、email、created_at

### 数据集（Dataset）
- id、user_id、name、total_rows、columns（JSON）、status、created_at

### 数据行（DataRow）
- id、task_id、row_index、original_data（JSON）、label_results（JSON）、ai_confidence（JSON）

### 标签（Label）
- id、user_id、name、version、scope、type、description、preprocessing_mode、extractor_config（JSON）

### 分析任务（AnalysisTask）
- id、dataset_id、user_id、name、status、total_rows、processed_rows、concurrency、model_config_id

### 标签结果（LabelResult）
- id、data_row_id、analysis_task_id、label_id、result、ai_confidence、ai_reasoning

### 大模型配置（ModelConfig）
- id、provider、api_key_encrypted、base_url、model、temperature、is_default

### 系统提示词（SystemPrompt）
- id、user_id、name、code、prompt_type、template（JSON）、variables（JSON）、is_system_default

### 提取器配置（ExtractorConfig）
- id、user_id、name、code、category、is_system

### 同步配置（SyncConfig）
- id、user_id、db_type、host、port、database_name、table_name、field_mappings（JSON）

### 执行日志（TaskExecutionLog）
- id、analysis_task_id、row_index、log_level、message、duration_ms

### 审计日志（AuditLog）
- id、user_id、action、resource_type、resource_id、details（JSON）

## 技术特性

1. **异步任务处理** - 使用 @Async + ThreadPoolTaskExecutor
2. **WebSocket 实时推送** - 任务进度、日志实时更新
3. **数据隔离** - 普通用户数据完全隔离
4. **版本管理** - 标签修改不覆盖历史版本
5. **信心度机制** - AI 判断结果附带信心度分数
6. **人工审核** - 低信心度结果标记为需人工审核
7. **二次强化分析** - 可配置触发条件对低信心度结果二次分析
8. **提示词模板引擎** - 支持变量替换、动态生成
9. **多数据库支持** - 支持连接外部数据库导入数据
10. **数据导出** - Excel/CSV 格式导出
