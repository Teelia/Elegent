# 涉警当事人信息完整性检查_v1 - 优化方案

## 一、背景与约束条件

### 1.1 当前问题总结
- 28条错误判定中，64%是因为"只找到一个身份证就判定是"，忽略了多个当事人
- 14%是因为身份证号前缀字符未检测
- 18%是因为公司主体未被识别

### 1.2 约束条件
| 约束 | 说明 |
|-----|------|
| **模型能力** | DeepSeek 70B（相比满血版推理能力有限） |
| **部署环境** | 内网部署，无法调用外部API |
| **响应时间** | 需考虑大模型调用的时间成本 |
| **资源限制** | 内网GPU资源有限 |

### 1.3 优化原则
1. **规则优先**：能用规则解决的问题，不依赖LLM
2. **防呆设计**：后置规则验证作为最终防线
3. **精确提取**：改进身份证号提取逻辑，避免误提取
4. **上下文理解**：利用LLM理解当事人关系，但结果需规则校验

## 二、优化方案（分阶段）

### 第一阶段：规则层优化（高优先级，1-2天）

#### 2.1 改进身份证号提取逻辑

**问题**：当前正则 `\d{15,19}[0-9Xx]?` 会匹配到带前缀的号码

**修改 `IdCardLengthValidator.java`**：

```java
// 修改 extractAndValidate 方法
public static ExtractResult extractAndValidate(String text, boolean allowLegacy) {
    ExtractResult result = new ExtractResult();

    // 新正则：匹配可能带前缀的身份证号
    // 识别模式：前缀字符 + 18位数字
    java.util.regex.Pattern prefixPattern =
        java.util.regex.Pattern.compile("[?？#＊\\s]{0,3}(\\d{17}[\\dXx])");
    java.util.regex.Matcher prefixMatcher = prefixPattern.matcher(text);

    while (prefixMatcher.find()) {
        String rawId = prefixMatcher.group(1);  // 带前缀的原始内容
        String cleanId = prefixMatcher.group(1).replaceAll("[^0-9Xx]", "");  // 清理后

        ExtractedItem item = new ExtractedItem();
        item.setValue(rawId);
        item.setRawValue(rawId);  // 保存原始值用于前缀检测

        // 检查是否有前缀字符
        if (rawId.length() > 18) {
            item.setValid(false);
            item.setReason("身份证号前存在前缀字符，导致格式错误");
        } else {
            // 正常验证
            ValidationResult vr = validate(cleanId, allowLegacy);
            item.setValid(vr.isValid());
            item.setReason(vr.getReason());
        }

        result.getItems().add(item);
        // ... 统计更新
    }

    return result;
}
```

**测试用例**：
```
输入: "？340811197505046323" → 输出: 无效，前缀字符
输入: "340811197505046323"   → 输出: 有效
输入: "34030219790910125X"   → 输出: 有效
```

#### 2.2 添加护照号验证

**新建 `PassportValidator.java`**：

```java
@Component
public class PassportValidator {

    // 中国护照：G/E/P + 8位数字
    private static final Pattern CN_PASSPORT =
        Pattern.compile("\\b[GEPM]\\d{8}\\b");

    // 外籍护照：1-2个大写字母 + 6-9位数字
    private static final Pattern FOREIGN_PASSPORT =
        Pattern.compile("\\b[A-Z]{1,2}\\d{6,9}\\b");

    public ValidationResult validate(String passport) {
        if (passport == null || passport.trim().isEmpty()) {
            return ValidationResult.invalid("护照号为空");
        }

        String trimmed = passport.trim();

        // 检查中国护照
        if (CN_PASSPORT.matcher(trimmed).matches()) {
            return ValidationResult.valid("中国护照格式正确");
        }

        // 检查外籍护照（更宽松的校验）
        if (FOREIGN_PASSPORT.matcher(trimmed).matches()) {
            return ValidationResult.valid("外籍护照格式正确");
        }

        return ValidationResult.invalid(
            String.format("护照号格式错误：%s（应为G/E/P+8位数字或其他标准格式）", trimmed));
    }

    public ExtractResult extractAndValidate(String text) {
        ExtractResult result = new ExtractResult();

        // 合并匹配两种模式
        Pattern pattern = Pattern.compile(
            "\\b[GEPM]\\d{8}\\b|\\b[A-Z]{1,2}\\d{6,9}\\b");
        Matcher matcher = pattern.matcher(text);

        while (matcher.find()) {
            String passport = matcher.group();
            ValidationResult vr = validate(passport);

            ExtractedItem item = new ExtractedItem();
            item.setValue(passport);
            item.setValid(vr.isValid());
            item.setReason(vr.getReason());

            result.getItems().add(item);
            if (vr.isValid()) {
                result.setValidCount(result.getValidCount() + 1);
            } else {
                result.setInvalidCount(result.getInvalidCount() + 1);
            }
        }

        return result;
    }
}
```

#### 2.3 添加公司主体识别

**修改 `PostProcessValidator.java`**：

```java
@Component
public class PostProcessValidator {

    // 公司主体关键词
    private static final List<String> COMPANY_KEYWORDS = Arrays.asList(
        "有限公司", "有限责任公司", "股份公司", "集团公司",
        "分公司", "合作社", "企业", "公司", "厂", "店", "铺"
    );

    // 统一社会信用代码正则（18位）
    private static final Pattern CREDIT_CODE =
        Pattern.compile("\\b[0-9A-Z]{18}\\b");

    // 营业执照注册号正则（15位数字）
    private static final Pattern REG_NUMBER =
        Pattern.compile("\\b\\d{15}\\b");

    /**
     * 检查公司主体是否有有效身份信息
     */
    public ValidationResult checkCompanyParty(String text) {
        // 1. 检测是否涉及公司主体
        List<String> detectedCompanies = detectCompanies(text);
        if (detectedCompanies.isEmpty()) {
            return ValidationResult.valid("未涉及公司主体");
        }

        // 2. 检查是否有营业执照信息
        boolean hasCreditCode = CREDIT_CODE.matcher(text).find();
        boolean hasRegNumber = REG_NUMBER.matcher(text).find();

        if (!hasCreditCode && !hasRegNumber) {
            return ValidationResult.invalid(
                String.format("涉及%d个公司主体，但未检测到营业执照信息：%s",
                    detectedCompanies.size(),
                    String.join("、", detectedCompanies.stream()
                        .limit(3)  // 只显示前3个
                        .collect(Collectors.toList()))));
        }

        return ValidationResult.valid("公司主体有有效身份信息");
    }

    private List<String> detectCompanies(String text) {
        List<String> companies = new ArrayList<>();
        for (String keyword : COMPANY_KEYWORDS) {
            // 简单检测：关键词 + 可能的名称
            Pattern pattern = Pattern.compile(
                "([\\u4e00-\\u9fa5]{2,10}" + keyword + ")");
            Matcher matcher = pattern.matcher(text);
            while (matcher.find()) {
                String company = matcher.group(1);
                if (!companies.contains(company)) {
                    companies.add(company);
                }
            }
        }
        return companies;
    }
}
```

### 第二阶段：当事人识别优化（中优先级，2-3天）

#### 2.4 改进当事人提取逻辑

**问题**：当前只提取"姓名（身份证号：xxx）"格式，遗漏了很多情况

**新增 `PartyExtractor.java`**：

```java
@Component
@Slf4j
public class PartyExtractor {

    // 报警人关键词
    private static final List<String> REPORTER_PATTERNS = Arrays.asList(
        "报警人", "报案人", "报称人", "当事人"
    );

    // 对方当事人关键词
    private static final List<String> OTHER_PARTY_PATTERNS = Arrays.asList(
        "对方", "对方当事人", "对方人员", "对方人员",
        "与.*发生", "与.*纠纷", "被.*打", "被.*撞",
        "对方车主", "对方司机", "对方用户", "对方客户"
    );

    /**
     * 提取所有涉警当事人
     *
     * @param text 原始文本
     * @param rowData 行数据（用于关联身份证号）
     * @return 提取的当事人列表
     */
    public List<Party> extractParties(String text, Map<String, Object> rowData) {
        List<Party> parties = new ArrayList<>();

        // 1. 提取报警人
        parties.addAll(extractReporter(text));

        // 2. 提取对方当事人
        parties.addAll(extractOtherParties(text));

        // 3. 关联身份信息
        associateIdCards(parties, text);

        return parties;
    }

    private List<Party> extractReporter(String text) {
        List<Party> parties = new ArrayList<>();

        for (String pattern : REPORTER_PATTERNS) {
            // 匹配模式："报警人：XXX" 或 "报警人XXX" 或 "报警人XXX（身份证号：xxx）"
            Pattern p = Pattern.compile(
                pattern + "[：:\\s]*([\\u4e00-\\u9fa5]{2,4})[（(]?");
            Matcher m = p.matcher(text);
            while (m.find()) {
                String name = m.group(1);
                parties.add(Party.builder()
                    .name(name)
                    .type(PartyType.REPORTER)
                    .build());
            }
        }

        return parties;
    }

    private List<Party> extractOtherParties(String text) {
        List<Party> parties = new ArrayList<>();

        // 提取具体人名（通常在对方当事人描述中）
        // 模式：姓名 +（身份证号/手机号/年龄）
        Pattern personPattern = Pattern.compile(
            "([\\u4e00-\\u9fa5]{2,4})[（(]\\s*(身份证|手机号|电话号码|年龄)");
        Matcher m = personPattern.matcher(text);
        while (m.find()) {
            String name = m.group(1);
            parties.add(Party.builder()
                .name(name)
                .type(PartyType.OTHER)
                .build());
        }

        // 提取"对方"泛指
        if (text.contains("对方当事人") || text.contains("对方人员")) {
            parties.add(Party.builder()
                .name("对方当事人")
                .type(PartyType.OTHER)
                .isGeneric(true)  // 泛指，需要进一步确认
                .build());
        }

        return parties;
    }

    private void associateIdCards(List<Party> parties, String text) {
        // 提取所有有效身份证号
        IdCardLengthValidator.ExtractResult idResult =
            IdCardLengthValidator.extractAndValidate(text, false);

        // 简单关联：按出现顺序，前N个身份证对应前N个当事人
        // 注意：这是简化实现，实际可能需要更精确的上下文分析
        int idIndex = 0;
        for (Party party : parties) {
            if (idIndex < idResult.getItems().size()) {
                IdCardLengthValidator.ExtractedItem idItem =
                    idResult.getItems().get(idIndex);
                if (idItem.isValid()) {
                    party.setIdCard(idItem.getValue());
                    party.setIdCardValid(true);
                }
                idIndex++;
            }
        }
    }
}

/**
 * 当事人
 */
@Data
@Builder
public class Party {
    private String name;           // 姓名
    private PartyType type;        // 类型
    private boolean isGeneric;     // 是否泛指（如"对方当事人"）
    private String idCard;         // 身份证号
    private boolean idCardValid;   // 身份证是否有效
}

/**
 * 当事人类型
 */
public enum PartyType {
    REPORTER,    // 报警人
    OTHER        // 对方当事人
}
```

#### 2.5 修改后置验证器，整合当事人检查

**修改 `PostProcessValidator.java`**：

```java
@Component
@RequiredArgsConstructor
public class PostProcessValidator {

    private final PartyExtractor partyExtractor;
    private final PassportValidator passportValidator;

    // ... 现有代码 ...

    /**
     * 验证当事人信息完整性（核心逻辑）
     */
    public ValidationResult validateParties(String text,
                                            Map<String, Object> rowData,
                                            List<ExtractedNumber> extractedNumbers) {

        // 1. 提取所有当事人
        List<Party> parties = partyExtractor.extractParties(text, rowData);

        if (parties.isEmpty()) {
            return ValidationResult.warning("未能识别到涉警当事人");
        }

        // 2. 统计有效证件数量
        int validIdCards = 0;
        int validPassports = 0;
        int partiesWithoutId = 0;

        for (Party party : parties) {
            if (party.isGeneric()) {
                // 泛指当事人：假设没有身份信息
                partiesWithoutId++;
                continue;
            }

            if (party.isIdCardValid()) {
                validIdCards++;
            } else {
                partiesWithoutId++;
            }
        }

        // 3. 护照号补充验证
        PassportValidator.ExtractResult passportResult =
            passportValidator.extractAndValidate(text);
        validPassports = passportResult.getValidCount();

        int totalValidIds = validIdCards + validPassports;
        int totalParties = parties.size();

        // 4. 判定结果
        if (partiesWithoutId > 0) {
            return ValidationResult.invalid(
                String.format("涉警当事人共%d人，但仅有%d人提供有效身份信息，缺失%d人",
                    totalParties, totalValidIds, partiesWithoutId));
        }

        if (totalValidIds < totalParties) {
            return ValidationResult.invalid(
                String.format("涉警当事人共%d人，有效证件仅%d个，信息不完整",
                    totalParties, totalValidIds));
        }

        return ValidationResult.valid(
            String.format("涉警当事人共%d人，身份信息完整", totalParties));
    }
}
```

### 第三阶段：LLM提示词优化（低优先级，1-2天）

#### 2.6 优化系统提示词

**考虑到70B模型能力限制，提示词需要更精确**：

```markdown
## 任务：判断涉警当事人信息是否完整

### 标签定义
检查警情中涉警当事人身份信息是否完整（身份证号/护照号）。

### 涉警当事人范围
- **纳入**：纠纷双方、侵权行为人、受害人等与警情事实直接相关人员
- **排除**：处警民警/辅警、围观群众、与警情无关路人

### 合格标准
所有涉警当事人均应提供有效身份号码：
- 18位身份证号（格式正确）
- 或护照号（G/E/P+8位数字）

### 直接合格情形（可跳过当事人检查）
若文本包含"撤警"、"无效警情"、"无事实"等关键词，可直接判定为"是"。

### 判定步骤（请严格按此顺序）：

**第一步：判断是否属于直接合格情形**
- 检查是否包含"请求撤警"、"误报警"、"重复警情"、"无效警情"、"未发现报警情况"等关键词
- 如果是，直接返回"是"，无需继续检查

**第二步：识别涉警当事人**
从文本中识别所有涉警当事人（纠纷双方、受害人等），记录姓名。

**第三步：检查身份信息**
对每个当事人，检查是否提供：
- 18位身份证号（格式正确）
- 或护照号

**第四步：综合判定**
- 如果任一当事人缺少有效身份信息 → 返回"否"
- 如果所有当事人都有效身份信息 → 返回"是"

### 输出格式（JSON）
```json
{
  "conclusion": "是" 或 "否",
  "confidence": 1-10,
  "reasoning": "判定理由",
  "parties": [
    {"name": "当事人姓名", "has_id": true/false, "id_type": "身份证/护照/无"}
  ],
  "is_withdrawal": true/false  // 是否属于撤警情形
}
```

### 重要提示
- 即使找到一个身份证号，如果还有其他当事人，必须检查所有当事人
- 身份证号前有`？`等字符也属于格式错误
- 公司作为当事人需要检查营业执照信息
```

## 三、集成方案

### 3.1 修改 EnhancementService

```java
@Service
@RequiredArgsConstructor
public class EnhancementService {

    private final PostProcessValidator postProcessValidator;
    // ... 其他依赖

    private EnhancementResult performPostValidation(...) {
        try {
            // 1. 先执行原有的后置验证
            PostProcessValidator.ValidationResult basicValidation =
                postProcessValidator.validateBasic(result, rowData, extractedNumbers);

            // 2. 新增：当事人信息完整性验证
            String text = extractText(rowData);
            PostProcessValidator.ValidationResult partyValidation =
                postProcessValidator.validateParties(text, rowData, extractedNumbers);

            // 3. 新增：公司主体验证
            PostProcessValidator.ValidationResult companyValidation =
                postProcessValidator.checkCompanyParty(text);

            // 汇总所有验证结果
            List<String> errors = new ArrayList<>();
            if (!basicValidation.isValid()) {
                errors.add(basicValidation.getMessage());
            }
            if (!partyValidation.isValid()) {
                errors.add(partyValidation.getMessage());
            }
            if (!companyValidation.isValid()) {
                errors.add(companyValidation.getMessage());
            }

            if (!errors.isEmpty()) {
                // 任一验证失败，修正为"否"
                return EnhancementResult.builder()
                    .finalResult("否")
                    .finalConfidence(Math.min(confidence, 50))
                    .validationNotes("规则验证失败: " + String.join("; ", errors))
                    .wasAdjusted(true)
                    .adjustmentReason("后置规则验证检测到问题，强制修正为[否]")
                    .rounds(0)
                    .build();
            }

            // ... 通过验证，返回原结果

        } catch (Exception e) {
            log.error("后置验证失败", e);
            // 验证失败时保守处理：维持原判但记录警告
            return EnhancementResult.builder()
                .finalResult(result)
                .finalConfidence(confidence)
                .validationNotes("后置验证异常: " + e.getMessage())
                .wasAdjusted(false)
                .rounds(0)
                .build();
        }
    }
}
