### 步骤 3.2: 增加强制触发条件

**目标文件**: `EnhancementConfig.java`

**新增方法**:
```java
/**
 * 扩展的触发条件判断
 * 增加强制二次验证的场景
 */
public boolean shouldTrigger(int initialConfidence, String validationResult) {
    // 原有触发条件
    if (initialConfidence < triggerConfidence) {
        return true;
    }

    // ✅ 新增：强制二次验证条件
    if (validationResult != null && !validationResult.isEmpty()) {
        // 存在格式错误的身份证号
        if (validationResult.contains("格式错误") ||
            validationResult.contains("长度错误") ||
            validationResult.contains("17位") ||
            validationResult.contains("19位") ||
            validationResult.contains("15位")) {
            log.debug("检测到身份证号格式异常，强制触发二次验证");
            return true;
        }

        // 存在信息缺失关键词
        if (validationResult.contains("拒绝") ||
            validationResult.contains("未提供") ||
            validationResult.contains("未登记") ||
            validationResult.contains("研判无果") ||
            validationResult.contains("无果")) {
            log.debug("检测到信息缺失关键词，强制触发二次验证");
            return true;
        }
    }

    return false;
}
```

## Phase 4: 质量保障机制（优先级：中）

### 步骤 4.1: 集成后置验证器

✅ **已创建** - [PostProcessValidator.java](backend/src/main/java/com/datalabeling/util/PostProcessValidator.java)

**集成位置**: 在 `DeepSeekService` 或 `AnalysisTaskAsyncService` 中调用

**示例集成代码**:
```java
@Autowired
private PostProcessValidator postProcessValidator;

// 在获得LLM判断结果后
LabelJudgeResult llmResult = callDeepSeek(...);

// 执行后置验证
PostProcessValidator.ValidationResult validationResult =
    postProcessValidator.validate(
        llmResult.getResult(),
        rowData,
        extractedNumbers
    );

// 根据验证结果调整最终输出
if (!validationResult.isValid()) {
    log.warn("后置验证失败，修正判断结果: {}", validationResult.getMessage());
    // 可以选择：1. 强制修正为"否" 2. 标记为需人工复核
    llmResult.setResult("否");
    llmResult.setConfidence(Math.min(llmResult.getConfidence(), 50));
    llmResult.addReason("后置规则验证: " + validationResult.getMessage());
}
```

## 常见问题

### Q1: 修改后标签版本不生效？

**A**: 检查数据库中的 `is_active` 字段，确保新版本为 true：

```sql
SELECT id, version, is_active FROM labels
WHERE name = '涉警当事人信息完整性检查';
```

### Q2: 二次强化没有触发？

**A**: 检查 `enhancement_config` 中的 `triggerConfidence` 值：

```sql
SELECT enhancement_config FROM labels
WHERE name = '涉警当事人信息完整性检查';
```

应该是 `{"triggerConfidence": 60, ...}`

### Q3: 预处理配置没有更新？

**A**: 需要重启应用或重新初始化标签：

```bash
# 方式1: 重启应用
mvn spring-boot:run

# 方式2: 通过管理API触发重新初始化
curl -X POST http://localhost:8080/api/admin/labels/reinit
```
