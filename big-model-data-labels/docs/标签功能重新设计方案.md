## 1. 设计概述

### 1.1 现有问题分析

当前标签功能存在以下限制：

| 问题 | 描述 | 影响 |
|------|------|------|
| **单一模式** | 只支持大模型判断，无法利用规则匹配 | 成本高、速度慢 |
| **提示词硬编码** | 提示词写死在代码中，不易优化 | 灵活性差 |
| **无法测试** | 创建标签后无法预览效果 | 质量难保证 |
| **无内置规则** | 需要每次手动描述规则 | 效率低 |

### 1.2 V3.0 核心改进

基于**身份证号精确提取**的成功经验，重新设计标签功能：

```
┌─────────────────────────────────────────────────────────┐
│                    标签功能 V3.0                         │
├─────────────────────────────────────────────────────────┤
│  三种规则模式                                           │
│  💬 大模型模式    - 复杂语义理解，准确性高                │
│  ⚡ 规则匹配模式  - 预置规则，速度快，成本低              │
│  🔀 混合模式       - 规则预筛 + LLM确认，平衡性能与准确性 │
├─────────────────────────────────────────────────────────┤
│  四层验证机制（身份证号提取专用）                        │
│  正则初筛 → 格式验证 → 校验位 → DeepSeek 70B语义         │
├─────────────────────────────────────────────────────────┤
│  提示词模板化                                           │
│  可在页面上创建、编辑、测试提示词模板                    │
├─────────────────────────────────────────────────────────┤
│  实时测试功能                                           │
│  创建标签时可测试规则效果，预览提取结果                  │
└─────────────────────────────────────────────────────────┘
```

### 1.3 核心设计原则

1. **灵活性**：支持多种规则模式，适应不同场景
2. **可测试**：创建标签时可实时预览效果
3. **可优化**：提示词模板可迭代优化
4. **高性能**：规则优先，减少大模型调用
5. **易用性**：简化配置流程，智能推荐

## 2. 标签规则模式设计

### 2.1 三种规则模式

#### 模式一：大模型判断模式（💬 LLM）

**适用场景**：
- 复杂语义理解
- 模糊规则判断
- 需要推理能力的场景

**特点**：
- ✅ 准确性高（95%+）
- ✅ 理解上下文
- ❌ 成本较高
- ❌ 速度较慢

**配置项**：
```json
{
  "ruleType": "llm",
  "promptTemplateId": 5,
  "llmConfig": {
    "temperature": 0.1,
    "maxTokens": 200,
    "includeReasoning": true
  }
}
```

#### 模式二：规则匹配模式（⚡ Rule）

**适用场景**：
- 标准格式提取（身份证号、手机号、邮箱等）
- 明确的匹配规则
- 对准确性要求极高的场景

**特点**：
- ✅ 速度快（毫秒级）
- ✅ 成本极低
- ✅ 准确率高（98%+）
- ❌ 无法处理复杂语义

**配置项**：
```json
{
  "ruleType": "rule",
  "rules": [
    {
      "builtinRuleId": 10,
      "ruleName": "id_card_18_deepseek",
      "targetColumn": "content"
    }
  ],
  "logic": "AND"
}
```

#### 模式三：混合模式（🔀 Hybrid）

**适用场景**：
- 需要平衡速度和准确性
- 大数据量处理
- 对成本敏感的场景

**特点**：
- ✅ 速度快（本地规则预筛）
- ✅ 准确性高（LLM确认）
- ✅ 成本可控（减少90% API调用）
- ⚠️ 配置稍复杂

**配置项**：
```json
{
  "ruleType": "hybrid",
  "firstStage": {
    "type": "rule",
    "rules": [
      {
        "builtinRuleId": 10,
        "ruleName": "id_card_18_strict",
        "targetColumn": "content"
      }
    ],
    "logic": "OR"
  },
  "secondStage": {
    "type": "llm",
    "trigger": "first_pass",
    "promptTemplateId": 5,
    "llmConfig": {
      "temperature": 0.1,
      "maxTokens": 200
    }
  }
}
```

### 2.2 规则模式对比

| 特性 | LLM模式 | Rule模式 | Hybrid模式 |
|------|---------|----------|------------|
| **准确性** | 95% | 98%+ | 98%+ |
| **速度** | 慢（~2s/条） | 快（~10ms/条） | 中等（~100ms/条） |
| **成本** | 高（¥0.02/条） | 零 | 低（¥0.002/条） |
| **配置复杂度** | 简单 | 中等 | 较复杂 |
| **适用场景** | 复杂语义 | 标准格式 | 大数据量 |

### 2.3 触发条件设计（混合模式）

```typescript
type TriggerCondition =
  | 'first_pass'  // 第一阶段通过时执行LLM
  | 'first_fail'  // 第一阶段失败时执行LLM
  | 'always'      // 总是执行LLM
```

**使用建议**：
- `first_pass`：规则初筛准确率高，需要LLM进一步确认
- `first_fail`：规则可能漏判，用LLM补充
- `always`：最高准确性，所有数据都经过LLM确认

## 4. 后端实现

### 4.1 标签服务扩展

#### `LabelService.java` 扩展

```java
@Service
public class LabelService {

    @Autowired
    private PromptTemplateService promptTemplateService;

    @Autowired
    private BuiltinRuleService builtinRuleService;

    @Autowired
    private RuleEngineService ruleEngineService;

    /**
     * 创建标签（支持三种规则模式）
     */
    @Transactional
    public Label createLabel(CreateLabelRequest request) {
        // 1. 验证配置
        validateLabelConfig(request);

        // 2. 构建规则配置
        String ruleConfig = buildRuleConfig(request);

        // 3. 创建标签实体
        Label label = Label.builder()
            .userId(request.getUserId())
            .name(request.getName())
            .version(1)
            .type(request.getType())
            .ruleType(request.getRuleType())
            .description(request.getDescription())
            .focusColumns(request.getFocusColumns())
            .extractFields(request.getExtractFields())
            .promptTemplateId(request.getPromptTemplateId())
            .ruleConfig(ruleConfig)
            .isActive(true)
            .build();

        return labelRepository.save(label);
    }

    /**
     * 验证标签配置
     */
    private void validateLabelConfig(CreateLabelRequest request) {
        String ruleType = request.getRuleType();

        switch (ruleType) {
            case "llm":
                // LLM模式必须指定提示词模板
                if (request.getPromptTemplateId() == null) {
                    throw new IllegalArgumentException("LLM模式必须选择提示词模板");
                }
                break;

            case "rule":
                // Rule模式必须指定至少一个规则
                if (request.getRules() == null || request.getRules().isEmpty()) {
                    throw new IllegalArgumentException("Rule模式必须选择至少一个内置规则");
                }
                break;

            case "hybrid":
                // Hybrid模式两阶段配置都要完整
                if (request.getFirstStageRules() == null || request.getFirstStageRules().isEmpty()) {
                    throw new IllegalArgumentException("Hybrid模式第一阶段必须指定规则");
                }
                if (request.getPromptTemplateId() == null) {
                    throw new IllegalArgumentException("Hybrid模式第二阶段必须指定提示词模板");
                }
                break;

            default:
                throw new IllegalArgumentException("不支持的规则类型: " + ruleType);
        }
    }

    /**
     * 构建规则配置JSON
     */
    private String buildRuleConfig(CreateLabelRequest request) {
        String ruleType = request.getRuleType();
        Map<String, Object> config = new HashMap<>();

        switch (ruleType) {
            case "llm":
                config.put("ruleType", "llm");
                config.put("promptTemplateId", request.getPromptTemplateId());
                config.put("llmConfig", request.getLlmConfig());
                break;

            case "rule":
                config.put("ruleType", "rule");
                config.put("rules", request.getRules());
                config.put("logic", request.getLogic());
                break;

            case "hybrid":
                config.put("ruleType", "hybrid");

                // 第一阶段配置
                Map<String, Object> firstStage = new HashMap<>();
                firstStage.put("type", "rule");
                firstStage.put("rules", request.getFirstStageRules());
                firstStage.put("logic", request.getFirstStageLogic());
                config.put("firstStage", firstStage);

                // 第二阶段配置
                Map<String, Object> secondStage = new HashMap<>();
                secondStage.put("type", "llm");
                secondStage.put("trigger", request.getHybridTrigger());
                secondStage.put("promptTemplateId", request.getPromptTemplateId());
                secondStage.put("llmConfig", request.getLlmConfig());
                config.put("secondStage", secondStage);
                break;
        }

        return objectMapper.writeValueAsString(config);
    }

    /**
     * 测试标签（在创建前预览效果）
     */
    public LabelTestResult testLabel(CreateLabelRequest request, Map<String, Object> testData) {
        try {
            // 1. 构建临时标签
            Label tempLabel = Label.builder()
                .name(request.getName())
                .type(request.getType())
                .ruleType(request.getRuleType())
                .description(request.getDescription())
                .focusColumns(request.getFocusColumns())
                .extractFields(request.getExtractFields())
                .ruleConfig(buildRuleConfig(request))
                .promptTemplateId(request.getPromptTemplateId())
                .build();

            // 2. 执行标签规则
            LabelResult result = ruleEngineService.executeRule(tempLabel, testData);

            // 3. 返回测试结果
            return LabelTestResult.builder()
                .success(true)
                .match(result.isMatch())
                .confidence(result.getConfidence())
                .extractedData(result.getExtractedData())
                .reasoning(result.getReasoning())
                .validationSteps(result.getValidationSteps())
                .processingTime(result.getProcessingTime())
                .build();

        } catch (Exception e) {
            return LabelTestResult.builder()
                .success(false)
                .error(e.getMessage())
                .build();
        }
    }
}
```

### 4.2 规则引擎服务（核心）

#### `RuleEngineService.java`

```java
@Service
public class RuleEngineService {

    @Autowired
    private DeepSeekService deepSeekService;

    @Autowired
    private BuiltinRuleService builtinRuleService;

    @Autowired
    private PromptTemplateService promptTemplateService;

    /**
     * 执行标签规则（路由到不同的执行器）
     */
    public LabelResult executeRule(Label label, Map<String, Object> rowData) {
        String ruleType = label.getRuleType();

        long startTime = System.currentTimeMillis();

        LabelResult result;
        switch (ruleType) {
            case "llm":
                result = executeLLMMode(label, rowData);
                break;
            case "rule":
                result = executeRuleMode(label, rowData);
                break;
            case "hybrid":
                result = executeHybridMode(label, rowData);
                break;
            default:
                throw new UnsupportedOperationException("不支持的规则类型: " + ruleType);
        }

        // 记录处理时间
        result.setProcessingTime(System.currentTimeMillis() - startTime);

        return result;
    }

    /**
     * 执行LLM模式
     */
    private LabelResult executeLLMMode(Label label, Map<String, Object> rowData) {
        // 1. 获取提示词模板
        PromptTemplate template = promptTemplateService.getTemplateById(label.getPromptTemplateId());

        // 2. 构建提示词参数
        Map<String, Object> params = buildPromptParams(label, rowData);

        // 3. 调用DeepSeek
        String prompt = promptTemplateService.buildPrompt(template.getId(), params);
        String response = deepSeekService.chat(prompt, template);

        // 4. 解析结果
        return parseLLMResponse(response, label.getType());
    }

    /**
     * 执行Rule模式
     */
    private LabelResult executeRuleMode(Label label, Map<String, Object> rowData) {
        // 1. 解析规则配置
        JSONObject ruleConfig = JSON.parseObject(label.getRuleConfig());
        JSONArray rules = ruleConfig.getJSONArray("rules");
        String logic = ruleConfig.getString("logic");

        // 2. 执行所有规则
        List<Boolean> ruleResults = new ArrayList<>();
        for (int i = 0; i < rules.size(); i++) {
            JSONObject ruleObj = rules.getJSONObject(i);
            boolean matched = executeBuiltinRule(ruleObj, rowData);
            ruleResults.add(matched);
        }

        // 3. 根据逻辑计算最终结果
        boolean finalMatch = "AND".equals(logic)
            ? ruleResults.stream().allMatch(r -> r)
            : ruleResults.stream().anyMatch(r -> r);

        return LabelResult.builder()
            .match(finalMatch)
            .method("rule")
            .confidence(finalMatch ? 1.0f : 0.0f)
            .validationSteps(ruleResults.stream()
                .map(r -> r ? "✅ 规则匹配通过" : "❌ 规则匹配失败")
                .collect(Collectors.toList()))
            .build();
    }

    /**
     * 执行Hybrid模式
     */
    private LabelResult executeHybridMode(Label label, Map<String, Object> rowData) {
        // 1. 解析配置
        JSONObject ruleConfig = JSON.parseObject(label.getRuleConfig());
        JSONObject firstStageConfig = ruleConfig.getJSONObject("firstStage");
        JSONObject secondStageConfig = ruleConfig.getJSONObject("secondStage");
        String trigger = secondStageConfig.getString("trigger");

        // 2. 第一阶段：规则匹配
        LabelResult firstStageResult = executeRuleMode(label, rowData);

        // 3. 判断是否需要执行第二阶段
        boolean needSecondStage = false;
        if ("always".equals(trigger)) {
            needSecondStage = true;
        } else if ("first_pass".equals(trigger)) {
            needSecondStage = firstStageResult.isMatch();
        } else if ("first_fail".equals(trigger)) {
            needSecondStage = !firstStageResult.isMatch();
        }

        LabelResult finalResult;
        if (needSecondStage) {
            // 第二阶段：LLM确认
            LabelResult secondStageResult = executeLLMMode(label, rowData);
            finalResult = secondStageResult;

            // 合并验证步骤
            List<String> allSteps = new ArrayList<>();
            allSteps.add("=== 第一阶段：规则匹配 ===");
            allSteps.addAll(firstStageResult.getValidationSteps());
            allSteps.add("=== 第二阶段：LLM确认 ===");
            allSteps.addAll(secondStageResult.getValidationSteps());
            finalResult.setValidationSteps(allSteps);

        } else {
            finalResult = firstStageResult;
        }

        return finalResult;
    }

    /**
     * 执行单个内置规则
     */
    private boolean executeBuiltinRule(JSONObject ruleObj, Map<String, Object> rowData) {
        Integer builtinRuleId = ruleObj.getInteger("builtinRuleId");
        String targetColumn = ruleObj.getString("targetColumn");

        // 获取内置规则
        BuiltinRule rule = builtinRuleService.getById(builtinRuleId);

        // 获取目标值
        Object targetValue = rowData.get(targetColumn);
        if (targetValue == null) {
            return false;
        }

        // 根据规则类型执行
        return builtinRuleService.executeRule(rule, String.valueOf(targetValue));
    }
}
```

## 6. 交互流程设计

### 6.1 创建标签流程

```
用户点击"新建标签"
    ↓
填写基础信息（名称、类型）
    ↓
选择规则模式（💬LLM / ⚡Rule / 🔀Hybrid）
    ↓
根据选择的模式，配置对应参数
    ├─ LLM模式：选择提示词模板、设置温度等参数
    ├─ Rule模式：选择内置规则、设置逻辑关系
    └─ Hybrid模式：配置两阶段参数
    ↓
输入测试数据（JSON格式）
    ↓
点击"运行测试"
    ↓
系统执行规则并返回结果
    ↓
用户查看测试结果
    ├─ 匹配/不匹配
    ├─ 置信度
    ├─ 提取内容
    ├─ 判断依据
    └─ 验证步骤
    ↓
如果不满意，调整配置后重新测试
    ↓
如果满意，点击"保存标签"
    ↓
标签创建成功，可在任务中使用
```

### 6.2 标签测试流程

```sequence
用户→前端: 点击"测试"按钮
前端→后端: 发送测试请求（包含标签配置+测试数据）
后端→规则引擎: 执行标签规则
规则引擎→DeepSeek: 调用大模型（如果需要）
DeepSeek→规则引擎: 返回结果
规则引擎→前端: 返回测试结果
前端→用户: 展示结果（匹配状态、置信度、验证步骤）
用户→前端: 查看结果，决定是否保存或调整配置
```
