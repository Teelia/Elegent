# 怎么做（How）

## 总体策略
围绕“涉警当事人信息完整性检查”的 **后置规则** 与 **二次强化提示词** 同步优化：

1) 规则侧：把“缺失关键词”从“词命中即死刑”改为“分级 + 上下文证据驱动”。  
2) 提示词侧：明确“缺失关键词仅在指向身份信息时才判否”，并要求输出可复核的缺失证据（不泄露完整证件号）。  
3) 口径侧：对“当事人范围/隐含当事人”做可配置化边界，避免规则与提示词互相放大误判。

## 规则修复方案（推荐）

### R1. 缺失关键词分级（Hard/Soft）
将现有缺失关键词拆分：

- **Hard（强制否）**：几乎可直接推断身份信息缺失  
  示例：匿名、拒绝提供身份证/身份信息、身份信息不详、无法核实身份、未获取到身份信息等
- **Soft（弱信号）**：需结合上下文（身份语义/当事人抽取）才能推翻  
  示例：“无法”“无果”等泛化词

实现要点：
- Soft 命中时，不直接 invalid；改为：
  - 若 PartyExtractor 判定存在缺失当事人 -> invalid（并输出缺失列表摘要）
  - 若证件号校验/提取存在明显问题 -> invalid
  - 否则 -> warning（可选：不改变结论，仅提示人工复核）

### R2. 缺失关键词“上下文判定”
对 Soft 类关键词引入上下文约束（建议用正则/窗口匹配）：
- 关键词附近（如 ±10~20 字符）是否出现：身份/身份证/证件/姓名/号码/登记/提供/核实 等身份语义
- 或整段文本是否出现 “无法查明/无法确定/无法获知 + 身份/姓名/证件” 等强模式

目标：避免 “无法联系/无法到达/无法打开App” 等非身份语义误杀。

### R3. 负例证据不足的纠偏策略（处理“其他”7行）
当 LLM 输出“否”，但规则侧检测：
- 身份证号均有效（或存在足够身份信息）
- 未命中 Hard 缺失关键词
- PartyExtractor 完整性为 complete

则给出策略开关（默认保守）：
- **保守模式**：返回 warning，不自动改结论（维持“否”，提示人工复核）
- **纠偏模式（可选）**：自动把结论修正为“是”，并记录“否->是（证据不足纠偏）”

## 提示词修复方案（推荐）

### P1. 纠正“缺失关键词=必否”的表述
在 `police_personnel_enhancement_v1` 中，将“出现上述关键词必须直接判否”改为：
- **仅当关键词明确指向身份信息缺失**（如身份/证件/登记/核实）才强制否
- 对泛化词（如“无法”）必须给出“身份语义证据”，否则不得推翻

### P2. 输出要求：缺失证据结构化且脱敏
强化输出字段（仍保持 JSON），新增/明确：
- `missing_parties`: 仅给角色/称谓（如“对方当事人/嫌疑人/女子/男子”），不要输出真实姓名
- `missing_fields`: e.g. ["id_card_or_passport"]
- `evidence`: 给出不包含完整证件号的证据短语（可遮蔽中间位）

### P3. 当事人范围口径提示
对“隐含当事人”给出更清晰的边界：
- 必须与警情事实直接相关且可识别为当事人
- 排除处警人员、仅提到的路人/围观者
- 对“泛指对方/一名男子”类：允许判否，但必须在理由中说明“无法识别身份信息”

## 数据库/配置落地方式
由于 `BuiltinGlobalLabelsInitializer` 只“补齐空字段”不覆盖存量配置，落地需走 DB/API 更新：

- 更新系统提示词（SystemPrompt）：`PUT /api/system-prompts/{id}` + `POST /api/system-prompts/{id}/refresh-cache`
- 更新标签（Label）：`PUT /api/labels/{id}`（创建新版本），同步更新：
  - label description（口径说明）
  - preprocessorConfig / extractorOptions（缺失关键词分级与上下文规则配置）
  - enhancementConfig（如需切换到新 prompt）

## 风险与回滚
- 风险：过度放松缺失关键词可能引入漏检（把真实缺失判为是）
- 回滚：保留旧标签版本（系统已版本化），随时可切回旧版本为 active

## ⚠️ 不确定因素（需要口径确认）
1) “涉警当事人”是否必须包含 **报警人/报案人**？还是仅纠纷双方/嫌疑人/受害人？  
2) 当文本仅出现“对方/一名男子/女子/嫌疑人”但无法提供身份信息时，是否**必须判否**（严格完整性）？  
3) 对“无法查明身份”但案件未进入处置阶段的记录，是否允许豁免（直接合格）？
