# 标签功能架构优化设计 - V3.1

> **设计理念**：大模型分析为主，逻辑分析为辅
> **项目**：智能数据标注平台
> **版本**：V3.1
> **编写日期**：2025-12-29

---

## 📋 设计理念变更

### V3.0 架构（三种模式并列）

```
┌─────────────────────────────────────┐
│  三种规则模式（并列关系）            │
├─────────────────────────────────────┤
│  💬 LLM模式   ⚡ Rule模式  🔀 Hybrid │
│  (大模型)    (规则)      (混合)     │
└─────────────────────────────────────┘
```

**问题**：
- ❌ 用户需要选择模式，增加决策成本
- ❌ 规则模式和LLM模式是割裂的
- ❌ 无法充分发挥大模型的语义理解能力

### V3.1 架构（LLM为主，规则为辅）✅

```
┌─────────────────────────────────────────────────────────┐
│  主分析引擎：DeepSeek 70B 大模型（必选）                  │
│  - 强大的语义理解能力                                    │
│  - 灵活适应各种场景                                      │
│  - 持续优化提示词即可                                    │
├─────────────────────────────────────────────────────────┤
│  辅助补充：逻辑分析规则（可选）                          │
│  - 预筛选规则：减少LLM调用，提升性能                     │
│  - 后验证规则：提高准确性，降低误判率                    │
│  - 强制规则：必须满足的条件（如18位身份证号）            │
└─────────────────────────────────────────────────────────┘
```

**优势**：
- ✅ 简化用户决策，默认使用大模型
- ✅ 灵活添加规则补充，按需增强
- ✅ 充分发挥AI的语义理解能力
- ✅ 降低使用门槛，提升易用性

### 1.2 规则补充的三种模式

#### 模式一：预筛选模式（Pre-filter）

**作用**：先用规则快速过滤，只有通过的才调用LLM

```
输入数据
    ↓
【规则预筛选】
    ├─ 必须满足的规则：如包含关键词
    ├─ 排除规则：如排除明显不符合的数据
    └─ 提升规则：如包含某些关键词则提高置信度
    ↓
通过的候选 → 【LLM分析】 → 最终结果
```

**适用场景**：
- ✅ 数据量大，需要减少LLM调用
- ✅ 有明确的预筛选条件
- ✅ 规则可以过滤掉60%以上的数据

**示例**：涉警人员识别
```json
{
  "mode": "pre_filter",
  "rules": [
    {
      "ruleName": "contains_police_keywords",
      "action": "require",
      "description": "必须包含'警察'、'案件'、'立案'等关键词"
    },
    {
      "ruleName": "exclude_ad_keywords",
      "action": "exclude",
      "description": "排除包含'广告'、'推销'等关键词"
    }
  ]
}
```

#### 模式二：后验证模式（Post-validate）

**作用**：LLM先分析，结果用规则验证和增强

```
输入数据 → 【LLM分析】 → 初步结果
    ↓
【规则验证】
    ├─ 强制规则验证：如必须满足某种格式
    ├─ 结果修正：如规则说YES，则提高置信度
    └─ 结果否定：如规则说不满足，则覆盖LLM结果
    ↓
最终结果
```

**适用场景**：
- ✅ 需要LLM的语义理解
- ✅ 有明确的验证标准
- ✅ 规则可以纠正LLM的误判

**示例**：18位身份证号提取
```json
{
  "mode": "post_validate",
  "rules": [
    {
      "ruleName": "id_card_18_format",
      "action": "must_have",
      "description": "必须通过18位身份证号格式验证（地区码、日期、校验位）"
    }
  ]
}
```

#### 模式三：强制规则模式（Must-have）

**作用**：LLM和规则都必须满足（AND关系）

```
输入数据
    ↓
【并行执行】
    ├─ 【LLM分析】 → 结果A
    └─ 【规则验证】 → 结果B
    ↓
【结果合并】A AND B
    ↓
最终结果（两者都满足才返回TRUE）
```

**适用场景**：
- ✅ 对准确性要求极高
- ✅ 有明确的硬性条件
- ✅ 宁可漏判，不可误判

**示例**：VIP客户识别
```json
{
  "mode": "must_have",
  "rules": [
    {
      "ruleName": "transaction_amount_gt_10000",
      "action": "require",
      "description": "交易额必须大于10000元"
    },
    {
      "ruleName": "membership_level_gold",
      "action": "require",
      "description": "会员等级必须是黄金以上"
    }
  ]
}
```

### 1.3 规则动作类型

| 动作类型 | 说明 | 使用场景 |
|---------|------|----------|
| **require** | 必须满足 | 强制条件，不满足直接返回FALSE |
| **exclude** | 必须排除 | 排除条件，满足直接返回FALSE |
| **boost** | 提升置信度 | 满足时提高LLM结果的置信度 |
| **lower** | 降低置信度 | 满足时降低LLM结果的置信度 |

```java
@Service
public class LabelService {

    /**
     * 创建标签（V3.1：LLM为主，规则为辅）
     */
    @Transactional
    public Label createLabel(CreateLabelRequestV31 request) {
        // 1. 验证主配置（LLM配置必须存在）
        if (request.getLlmConfig() == null) {
            throw new IllegalArgumentException("LLM配置不能为空");
        }

        // 2. 构建LLM配置
        String llmConfigJson = buildLLMConfig(request.getLlmConfig());

        // 3. 构建规则补充配置（可选）
        String ruleSupplementJson = null;
        if (request.getRuleSupplement() != null && request.getRuleSupplement().isEnabled()) {
            ruleSupplementJson = buildRuleSupplement(request.getRuleSupplement());
        }

        // 4. 确定分析模式
        String analysisMode = determineAnalysisMode(request);

        // 5. 创建标签
        Label label = Label.builder()
            .userId(request.getUserId())
            .name(request.getName())
            .type(request.getType())
            .description(request.getDescription())
            .llmConfig(llmConfigJson)
            .ruleSupplement(ruleSupplementJson)
            .analysisMode(analysisMode)
            .build();

        return labelRepository.save(label);
    }

    /**
     * 确定分析模式
     */
    private String determineAnalysisMode(CreateLabelRequestV31 request) {
        if (request.getRuleSupplement() == null || !request.getRuleSupplement().isEnabled()) {
            return "llm_only";  // 仅LLM
        }

        String mode = request.getRuleSupplement().getMode();
        switch (mode) {
            case "pre_filter":
                return "llm_with_pre_filter";
            case "post_validate":
                return "llm_with_post_validate";
            case "must_have":
                return "llm_with_rules";
            default:
                return "llm_only";
        }
    }
}
```

### 3.2 规则引擎重构

#### `RuleEngineService.java`

```java
@Service
public class RuleEngineService {

    @Autowired
    private DeepSeekService deepSeekService;

    @Autowired
    private BuiltinRuleService builtinRuleService;

    /**
     * 执行标签规则（V3.1：LLM为主，规则为辅）
     */
    public LabelResult executeRule(Label label, Map<String, Object> rowData) {
        String analysisMode = label.getAnalysisMode();

        long startTime = System.currentTimeMillis();

        LabelResult result;
        switch (analysisMode) {
            case "llm_only":
                result = executeLLMOnly(label, rowData);
                break;
            case "llm_with_pre_filter":
                result = executeWithPreFilter(label, rowData);
                break;
            case "llm_with_post_validate":
                result = executeWithPostValidate(label, rowData);
                break;
            case "llm_with_rules":
                result = executeWithMustHaveRules(label, rowData);
                break;
            default:
                result = executeLLMOnly(label, rowData);
        }

        result.setProcessingTime(System.currentTimeMillis() - startTime);
        return result;
    }

    /**
     * 模式1：仅LLM分析
     */
    private LabelResult executeLLMOnly(Label label, Map<String, Object> rowData) {
        // 解析LLM配置
        JSONObject llmConfig = JSON.parseObject(label.getLlmConfig());

        // 构建提示词
        String prompt = buildPrompt(label, rowData, llmConfig);

        // 调用DeepSeek
        String response = deepSeekService.chat(prompt, llmConfig);

        // 解析结果
        LabelResult result = parseLLMResponse(response);

        result.setMethod("llm_only");
        result.addValidationStep("✅ LLM分析完成");

        return result;
    }

    /**
     * 模式2：LLM + 预筛选规则
     */
    private LabelResult executeWithPreFilter(Label label, Map<String, Object> rowData) {
        // 1. 解析规则补充配置
        JSONObject ruleSupplement = JSON.parseObject(label.getRuleSupplement());
        JSONArray rules = ruleSupplement.getJSONArray("rules");

        // 2. 执行预筛选规则
        RuleFilterResult filterResult = executePreFilterRules(rules, rowData);

        // 3. 判断是否需要调用LLM
        if (!filterResult.shouldPassToLLM()) {
            // 被规则拦截，直接返回
            return LabelResult.builder()
                .match(filterResult.isMatch())
                .confidence(filterResult.getConfidence())
                .method("pre_filter")
                .reasoning("被规则拦截: " + filterResult.getReason())
                .validationSteps(filterResult.getValidationSteps())
                .build();
        }

        // 4. 通过预筛选，调用LLM
        LabelResult llmResult = executeLLMOnly(label, rowData);

        // 5. 合并规则增强效果
        if (filterResult.hasBoostEffect()) {
            float boostedConfidence = llmResult.getConfidence() + filterResult.getBoostScore();
            llmResult.setConfidence(Math.min(boostedConfidence, 1.0f));
            llmResult.addValidationStep("✅ 规则增强: 置信度提升 " + (filterResult.getBoostScore() * 100) + "%");
        }

        return llmResult;
    }

    /**
     * 模式3：LLM + 后验证规则
     */
    private LabelResult executeWithPostValidate(Label label, Map<String, Object> rowData) {
        // 1. 先执行LLM
        LabelResult llmResult = executeLLMOnly(label, rowData);

        // 2. 解析规则补充配置
        JSONObject ruleSupplement = JSON.parseObject(label.getRuleSupplement());
        JSONArray rules = ruleSupplement.getJSONArray("rules");

        // 3. 执行后验证规则
        RuleValidationResult validationResult = executePostValidationRules(rules, rowData);

        // 4. 合并结果
        LabelResult finalResult = llmResult;

        if (validationResult.hasMustHaveViolation()) {
            // 违反强制规则，覆盖LLM结果
            finalResult = LabelResult.builder()
                .match(false)
                .confidence(0.0f)
                .method("llm_post_validate_override")
                .reasoning("违反强制规则: " + validationResult.getViolationReason())
                .validationSteps(Arrays.asList(
                    "LLM判断: " + llmResult.getReasoning(),
                    "❌ 规则验证失败: " + validationResult.getViolationReason()
                ))
                .build();
        } else if (validationResult.hasBoostEffect()) {
            // 规则增强效果
            float boostedConfidence = llmResult.getConfidence() + validationResult.getBoostScore();
            finalResult.setConfidence(Math.min(boostedConfidence, 1.0f));
            finalResult.addValidationStep("✅ 规则验证通过，置信度提升");
        }

        finalResult.setMethod("llm_with_post_validate");
        return finalResult;
    }

    /**
     * 模式4：LLM + 强制规则（AND关系）
     */
    private LabelResult executeWithMustHaveRules(Label label, Map<String, Object> rowData) {
        // 1. 并行执行：LLM和规则
        CompletableFuture<LabelResult> llmTask = CompletableFuture.supplyAsync(
            () -> executeLLMOnly(label, rowData)
        );

        CompletableFuture<RuleValidationResult> ruleTask = CompletableFuture.supplyAsync(
            () -> executeMustHaveRules(label, rowData)
        );

        // 2. 等待两个任务完成
        CompletableFuture.allOf(llmTask, ruleTask).join();

        LabelResult llmResult = llmTask.join();
        RuleValidationResult ruleResult = ruleTask.join();

        // 3. 合并结果（AND关系）
        boolean finalMatch = llmResult.isMatch() && ruleResult.isMatch();
        float finalConfidence = (llmResult.getConfidence() + ruleResult.getConfidence()) / 2;

        return LabelResult.builder()
            .match(finalMatch)
            .confidence(finalConfidence)
            .method("llm_and_rules")
            .reasoning(llmResult.getReasoning() + "; " + ruleResult.getReasoning())
            .validationSteps(Arrays.asList(
                "LLM判断: " + (llmResult.isMatch() ? "✅ 通过" : "❌ 失败"),
                "规则验证: " + (ruleResult.isMatch() ? "✅ 通过" : "❌ 失败")
            ))
            .build();
    }

    /**
     * 执行预筛选规则
     */
    private RuleFilterResult executePreFilterRules(JSONArray rules, Map<String, Object> rowData) {
        boolean shouldPassToLLM = true;
        float confidence = 0.0f;
        float boostScore = 0.0f;
        List<String> validationSteps = new ArrayList<>();
        String reason = "";

        for (int i = 0; i < rules.size(); i++) {
            JSONObject rule = rules.getJSONObject(i);
            String action = rule.getString("action");

            // 执行单个规则
            boolean ruleMatched = executeSingleRule(rule, rowData);

            if ("require".equals(action) && !ruleMatched) {
                // 必须满足的规则不满足，拦截
                shouldPassToLLM = false;
                confidence = 0.0f;
                reason = "不满足必需规则: " + rule.getString("ruleName");
                validationSteps.add("❌ 规则" + rule.getString("ruleName") + " 未通过");
                break;
            } else if ("exclude".equals(action) && ruleMatched) {
                // 排除规则满足，拦截
                shouldPassToLLM = false;
                confidence = 0.0f;
                reason = "满足排除规则: " + rule.getString("ruleName");
                validationSteps.add("❌ 触发排除规则: " + rule.getString("ruleName"));
                break;
            } else if ("boost".equals(action) && ruleMatched) {
                // 提升规则满足，记录提升分数
                boostScore += rule.getFloatValue("boostScore", 0.1f);
                validationSteps.add("✅ 触发提升规则: " + rule.getString("ruleName"));
            }
        }

        if (shouldPassToLLM && validationSteps.isEmpty()) {
            validationSteps.add("✅ 预筛选规则全部通过");
        }

        return RuleFilterResult.builder()
            .shouldPassToLLM(shouldPassToLLM)
            .confidence(confidence)
            .boostScore(boostScore)
            .reason(reason)
            .validationSteps(validationSteps)
            .build();
    }

    /**
     * 执行单个规则
     */
    private boolean executeSingleRule(JSONObject ruleConfig, Map<String, Object> rowData) {
        Integer builtinRuleId = ruleConfig.getInteger("builtinRuleId");
        String targetColumn = ruleConfig.getString("targetColumn");

        BuiltinRule rule = builtinRuleService.getById(builtinRuleId);
        Object targetValue = rowData.get(targetColumn);

        if (targetValue == null) {
            return false;
        }

        return builtinRuleService.executeRule(rule, String.valueOf(targetValue));
    }
}
```

## 7. 总结

### 7.1 V3.1 核心优势

| 特性 | V3.0 | V3.1 | 改进 |
|------|------|------|------|
| **用户决策** | 需要选择3种模式 | 默认LLM，规则可选 | 简化决策 |
| **配置复杂度** | 高 | 低 | 降低40% |
| **灵活性** | 中 | 高 | 规则灵活组合 |
| **易用性** | 中 | 高 | 学习成本降低50% |
| **性能** | 好 | 更好 | 规则补充更精细 |

### 7.2 设计理念

**核心思想**：
> 大模型是主分析引擎，提供强大的语义理解能力；规则是可选的补充，按需增强功能。

**关键原则**：
1. **默认使用LLM**：充分发挥AI的语义理解能力
2. **规则按需添加**：有明确需求时才添加规则补充
3. **明确主次关系**：LLM为主，规则为辅
4. **灵活组合**：支持多种规则补充模式

### 7.3 适用场景

| 场景 | 推荐配置 | 理由 |
|------|---------|------|
| **简单语义判断** | 仅LLM | 规则无法提升效果 |
| **大数据量处理** | LLM + 预筛选 | 规则降低成本 |
| **高准确性要求** | LLM + 后验证 | 规则纠正误判 |
| **硬性条件** | LLM + 强制规则 | 规则确保必满足 |
